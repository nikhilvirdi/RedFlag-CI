RedFlag CI is a security intelligence platform built for AI-assisted software development. It plugs into GitHub pull requests, analyzes code changes before merge, and identifies both classic and AI-specific security vulnerabilities.

## The Problem

AI coding assistants make teams faster, but they also introduce a new category of security issues that conventional tools do not fully understand.

AI-generated code can include hardcoded credentials, hallucinated packages, disabled authentication scaffolding, insecure defaults, prompt injection paths, unsafe query construction, and development-only flags left in production paths. Traditional SAST tools detect the first category but miss or underweight the second.

## RedFlag CI

RedFlag CI is a GitHub App-based security system designed specifically for modern AI-assisted development workflows. It combines static analysis, semantic scanning, dependency intelligence, repository context, and LLM-assisted remediation into a single workflow.

The project focuses on pull-request-time detection so developers get feedback before insecure code is merged. It is built to be zero-configuration at onboarding while still supporting richer workflows for security teams and enterprises.

## What it does

RedFlag CI scans pull request diffs and detects both common security flaws and issues that are strongly associated with AI-generated code.

### Detection coverage

- Finds exposed secrets, credential patterns, and high-entropy probable secrets.
- Detects SQL injection, prompt injection, command injection, path traversal, SSRF, XSS, insecure deserialization, and related application-layer issues.
- Verifies whether referenced npm and PyPI packages are real to catch hallucinated dependencies.
- Flags insecure defaults such as wildcard CORS, disabled auth, debug flags, broad permissions, weak crypto, and missing validation.
- Scans Dockerfiles, Docker Compose files, and Kubernetes manifests for infrastructure misconfigurations.
- Checks dependency risk, typosquats, ghost dependencies, and license conflicts.
- Searches git history for deleted but still recoverable secrets.
- Compares new code against baseline memory and semantic embeddings to identify suspicious patterns with lower noise.
- Correlates multiple findings into exploit chains and computes a weighted security risk score.
- Produces contextual remediation and supports automatic fix pull requests where appropriate.

## How it does it

RedFlag CI follows an asynchronous review pipeline so pull request events are handled quickly while analysis runs in the background.

```text
Pull Request Event
  -> GitHub App webhook received
  -> Signature verification
  -> Job queued with BullMQ
  -> PR diff fetched from GitHub
  -> Parallel analyzers execute
  -> Findings aggregated
  -> Vulnerability chains identified
  -> Risk score computed
  -> Remediation generated
  -> PR comment and status updated
  -> Results stored for history and posture tracking
```

### Pipeline summary

- A GitHub App receives pull request events and passes the work into a Redis-backed BullMQ queue.
- An Express.js service orchestrates webhook handling, job management, authentication, persistence, and integration flows.
- A Python scan engine runs layered analyzers for source code, dependencies, infrastructure files, and git history.
- A chaining engine correlates findings into multi-step attack paths instead of reporting only isolated alerts.
- A remediation layer generates contextual explanations and fix suggestions, including automatic fix PRs for supported cases.
- PostgreSQL, Prisma, and pgvector store scans, findings, baselines, trends, and repository memory.
- A Next.js dashboard surfaces scan history, repository posture, and long-term risk trends.

## Features

- GitHub App integration with pull-request-first scanning.
- Zero-configuration onboarding for repository installation.
- AI-specific detection rules in addition to standard SAST-style coverage.
- Vulnerability chaining for realistic exploit-path analysis.
- LLM-powered remediation with context-aware fix guidance.
- Automatic fix pull request support for fixable findings.
- Baseline memory and false-positive reduction over time.
- Security posture scoring and trend tracking.
- Public rule registry and extensible detection ecosystem.
- SARIF export for compatibility with external tooling.
- Slack, Discord, and outbound webhook notifications.
- Audit logging and community-facing platform features.

## Why it is better than existing solutions

Most existing security tools are strong at finding known isolated patterns, but RedFlag CI is designed for the way AI-assisted code is actually produced and shipped.

### Advantages

- **AI-native focus:** It targets failure modes common in AI-generated code, including hallucinated packages, disabled scaffolding, and prompt-construction risks.
- **Exploit-chain reasoning:** It correlates multiple findings into realistic attack paths instead of presenting only disconnected alerts.
- **Repository memory:** It learns from repository history and baseline state so developers see more newly introduced risk and less repetitive noise.
- **Contextual remediation:** It provides code-aware fixes rather than relying only on generic static recommendations.
- **Fix workflow support:** It can move beyond alerts and generate auto-fix pull requests for supported issues.
- **Broader security surface:** It combines app-layer scanning, dependency analysis, IaC checks, git-history review, license risk detection, and semantic similarity analysis in one system.

## Tech Stack

### Backend and orchestration

| Technology | Category | Why this is used in the project |
|---|---|---|
| Node.js | Runtime | Runs the core backend services and supports the event-driven workflow needed for webhook handling and async orchestration. |
| Express.js | Backend framework | Provides a lightweight and flexible HTTP layer for webhooks, APIs, middleware, and service composition. |
| TypeScript | Language | Adds strong typing across backend and frontend code to reduce defects and improve maintainability. |
| Zod | Validation | Validates request payloads and runtime boundaries safely before data enters core services. |
| Axios | HTTP client | Handles outbound API calls to GitHub and the remediation provider. |
| Winston | Logging | Provides structured application logging for operational visibility and troubleshooting. |
| Morgan | Request logging | Captures HTTP request logs in a consistent format during API and webhook activity. |
| BullMQ | Job queue | Moves scan work into background jobs so pull request events can be handled efficiently. |
| Redis | Queue/cache store | Powers queue state, retries, counters, and other fast in-memory coordination needs. |
| ioredis | Redis client | Connects application services cleanly to Redis-backed queue and rate-limit workflows. |
| node-cron | Scheduler | Runs recurring operations such as scheduled full-repository scans. |
| GitHub OAuth 2.0 | Authentication | Supports developer sign-in and account association with GitHub identities. |
| JWT | Session/auth | Protects authenticated routes with token-based access control. |
| @octokit/app | GitHub integration | Handles GitHub App authentication and installation-level interactions. |
| @octokit/rest | GitHub integration | Supports GitHub API operations like pull request and repository actions. |

### Scan engine and analysis

| Technology | Category | Why this is used in the project |
|---|---|---|
| Python | Scan engine language | Powers the analyzer layer where pattern matching, heuristics, and code analysis are executed. |
| Python AST | Code parsing | Enables structure-aware analysis instead of relying only on plain-text matching. |
| Regex analyzers | Pattern detection | Provides fast targeted detection for known high-signal vulnerability patterns. |
| Semgrep | SAST engine | Expands static analysis coverage for common security issues across codebases. |
| TruffleHog | Secret history scanning | Detects secrets that may still exist in repository history even after removal from current code. |
| Checkov | IaC scanning | Analyzes infrastructure-as-code files for risky cloud and container misconfigurations. |
| npm Registry API | Package verification | Confirms whether JavaScript packages actually exist to catch hallucinated dependencies. |
| PyPI API | Package verification | Confirms whether Python packages exist and supports package legitimacy checks. |
| SPDX API/database | License intelligence | Helps identify risky, unknown, or conflicting dependency licenses. |
| Embedding model | Semantic analysis | Supports similarity-based detection for obfuscated or paraphrased vulnerable code. |
| pgvector | Vector search | Stores and queries code embeddings for repository memory and semantic comparison. |
